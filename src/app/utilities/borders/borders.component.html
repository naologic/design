Borders

Border based on grid breakpoints notation

Spacing utilities that apply to all breakpoints, from xs to xl, have no breakpoint abbreviation in them. This is because those classes are applied from min-width: 0 and up, and thus are not bound by a media query. The remaining breakpoints, however, do include a breakpoint abbreviation.

The classes are named using the format border-[sides]-[breakpoint]-[property] for xs, for sm, md, lg, and xl.

Where sides is one of:

  • top - for classes that set border-top
  • right - for classes that set border-bottom
  • bottom - for classes that set border-left
  • left - for classes that set border-right

Where breakpoint is one of:

  • xs - for classes that sets the border for devices > xs
  • sm - for classes that sets the border for devices > sm
  • md - for classes that sets the border for devices > md
  • lg - for classes that sets the border for devices > lg
  • xl - for classes that sets the border for devices > xl

Where property is one of:

  • 1 - for classes that sets the border to 1
  • 0 - for classes that sets the border to 0

All the .border-right-* .border-left-* classes are RTL ready (the borders are flipping on the RTL version, for example: .border-right-lg-1 would have border-left: 1 solid; insted of border-right: 1 solid;).

Borders custom colors

  • border-bg-1 - for classes that sets the color to var(--bg-1)
  • border-bg-2 - for classes that sets the color to var(--bg-2)
  • border-text-6 - for classes that sets the color to var(--text-6)
